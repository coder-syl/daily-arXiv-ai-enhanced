### [1] [Visual Text Compression as Measure Transport](https://arxiv.org/abs/2605.06708)
*Lv Tang,Tianyi Zheng,Yang Liu,Bo Li,Xingyu Li*

Main category: cs.CV

TL;DR: 本文提出一种基于测度传输理论的视觉文本压缩（VTC）分析框架，通过估计压缩过程中的信息损失，实现无需下游标签的路径选择与高成本区域的高分辨率重编码，在24个NLP数据集上显著提升性能并减少token使用。


<details>
  <summary>Details</summary>
Motivation: 现有视觉文本压缩方法虽能大幅减少token数量，但其在不同任务上的效果不稳定，且压缩比无法预测性能表现。根本问题在于缺乏对视觉编码所导致的任务相关信息损失的量化方法。

Method: 将文本和视觉token视为经验概率测度，利用ViT patch编码器诱导的前推映射，将传输代价分解为patch内聚合的精度代价和跨patch碎片化的覆盖代价，并通过无标签探针估计这两项代价。基于此，提出无标签路由准则和基于传输代价的中央凹机制。

Result: 在Qwen3-4B模型和24个NLP数据集上，所提出的无标签路由规则在17个数据集上匹配最优选择（70.8%），相比纯LLM平均任务得分提升+3.3%，同时平均减少10.3%的token。

Conclusion: 通过测度传输视角可有效量化VTC中的信息损失，从而指导更优的压缩与推理策略，在保持效率的同时提升下游任务性能。

Abstract: Visual text compression (VTC) promises efficient long-context processing by rendering text into an image and re-encoding it with a vision-language model, often producing $3$--$20\times$ fewer decoder tokens than subword tokenization. Yet token savings do not translate predictably into downstream utility: on some tasks the visual path matches or exceeds the text path, on others it collapses, and the compression ratio itself does not predict which regime will occur. The missing quantity is therefore not another summary of efficiency, but a principled measure of task-relevant information loss induced by visual encoding. We address this problem by formulating VTC in the language of measure transport. Treating text and visual tokens as empirical probability measures, we show that the ViT patch encoder induces a push-forward map whose transport cost decomposes into a precision cost from within-patch aggregation and a coverage cost from cross-patch fragmentation. Both terms are estimable from downstream-label-free probes. This formulation yields two operational consequences: a downstream-label-free routing criterion that selects whether to use the visual path for a given input or benchmark instance, and a transport-informed foveation mechanism that re-encodes high-cost regions at higher resolution. Across $24$ NLP datasets at Qwen3-4B, our label-free rule matches the per-dataset oracle on $17/24$ datasets ($70.8\%$), and improves the average task score by $+3.3\%$ with $-10.3\%$ average tokens relative to a pure-LLM.

Abstract (中文翻译): 视觉文本压缩（VTC）通过将文本渲染为图像并使用视觉语言模型重新编码，有望实现高效的长上下文处理，通常比子词切分产生少3至20倍的解码器token。然而，token节省并不能可靠地转化为下游效用：在某些任务上，视觉路径的表现与文本路径相当甚至更优，而在其他任务上则完全失效，且压缩比本身无法预测哪种情况会发生。因此，当前缺失的并非另一种效率总结，而是对视觉编码所引入的任务相关信息损失的原理性度量。我们通过测度传输的语言来形式化VTC问题。将文本和视觉token视为经验概率测度，我们证明ViT图像块编码器诱导了一个前推映射，其传输代价可分解为来自块内聚合的精度代价和来自跨块碎片化的覆盖代价。这两项代价均可通过无需下游标签的探针进行估计。该形式化带来了两个实用成果：一是无需下游标签的路由准则，用于决定对给定输入或基准实例是否采用视觉路径；二是基于传输代价的中央凹机制，对高代价区域以更高分辨率重新编码。在Qwen3-4B模型上对24个NLP数据集的实验表明，我们的无标签规则在17/24个数据集（70.8%）上与逐数据集最优选择一致，并相比纯大语言模型平均提升任务得分3.3%，同时平均减少10.3%的token。

Method: 提出 AdpSplit 方法：基于 L1 像素误差区域统计，自适应地决定每次分裂产生的子高斯数量，并据此初始化其参数。

Experiments on real image graphs show that TriDE improves direction accuracy by a large margin and yields better downstream camera locations, providing a practical link between local pairwise estimation and global camera pose geometry.

Abstract (中文翻译): 扩散变换器（DiTs）在视频生成质量方面达到了最先进的水平，但其计算成本极高，因为标准推理对序列中的每个 token 统一应用相同数量的去噪步骤。众所周知，人类视觉会忽略大量冗余的运动信息。那么，为何我们最密集的模型却对每个时空 token 一视同仁？本文提出了异构步长分配（Heterogeneous Step Allocation, HSA），这是一种无需训练的推理算法，根据各时空 token 的速度动态为其分配不同的去噪步预算。为了解决由此产生的序列长度不匹配问题，同时不牺牲全局上下文，HSA 引入了一种 KV 缓存同步机制，使活跃 token 能够关注整个序列，同时完全绕过非活跃 token。此外，我们推导出一种缓存欧拉更新方法，可在单次操作中推进被跳过 token 的潜在状态，而无需额外的模型评估。我们在 Wan-2 和 LTX-2 模型上对文本到视频（T2V）和图像到视频（I2V）生成任务评估了 HSA。结果表明，HSA 显著优于先前最先进的缓存方法和原始 Flow Matching 基线，尤其是在激进的加速设置下（例如 50% 和 25% 的运行时间）。关键在于，HSA 在无需昂贵的离线性能分析的情况下，实现了更优的质量-运行时间帕累托前沿，即使在严格的计算预算下也能稳健地保持结构完整性和生成质量。
